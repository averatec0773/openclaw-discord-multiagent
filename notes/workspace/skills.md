# Skills & Workspace Files — Structure Reference

> Docs: https://docs.openclaw.ai/tools/skills

---

## Directory Structure (inside container)

```
/home/node/.openclaw/          ← config volume (persistent across rebuilds)
├── skills/                    ← managed skills (shared across all agents)
│   └── self-improving-agent/  ← installed via clawhub
│       └── SKILL.md
├── workspace/                 ← agent workspace
│   ├── AGENTS.md              ← agent behavior & session startup instructions
│   ├── TOOLS.md               ← environment-specific tool notes (SSH hosts, accounts, etc.)
│   ├── SOUL.md                ← agent personality & values
│   ├── MEMORY.md              ← persistent memory across sessions
│   ├── IDENTITY.md            ← auto-generated by OpenClaw
│   ├── USER.md                ← auto-generated user profile
│   ├── HEARTBEAT.md           ← auto-generated health info
│   ├── BOOTSTRAP.md           ← auto-generated bootstrap instructions
│   └── skills/                ← workspace-specific skills (currently empty)
└── openclaw.json              ← main config
```

Host path `/root/.openclaw/` maps to `/home/node/.openclaw/` inside container (bind mount).

---

## Skills Loading Precedence

1. **Workspace skills** — `<workspace>/skills/` (highest priority)
2. **Managed skills** — `~/.openclaw/skills/` (default location, used by clawhub)
3. **Bundled skills** — `/app/skills/` (shipped with image, read-only)
4. **extraDirs** — additional paths in `openclaw.json` (lowest priority)

Current `openclaw.json`:
```json
"skills": {
  "load": {
    "extraDirs": ["/home/node/.openclaw/skills"]
  }
}
```
> Note: `extraDirs` here points to the same path as managed skills — redundant but harmless.

## Skill File Format

Each skill is a directory containing `SKILL.md` with YAML frontmatter:

```markdown
---
name: skill-name
description: Brief description of what this skill does
---

Skill instructions and content here.
```

Optional frontmatter keys: `homepage`, `user-invocable`, `disable-model-invocation`, `command-dispatch`, `requires`

---

## Workspace Files — Purpose

| File | Purpose |
|---|---|
| `AGENTS.md` | Session startup sequence, agent behavior rules |
| `TOOLS.md` | Your specific setup: SSH aliases, API accounts, device names |
| `SOUL.md` | Agent personality, tone, values |
| `MEMORY.md` | Persistent notes the agent writes and reads across sessions |
| `IDENTITY.md` | Auto-managed by OpenClaw |
| `USER.md` | Auto-managed user profile |

> **TOOLS.md vs Skills**: Skills define *how* tools work (instructions for the LLM). TOOLS.md holds *your* specifics — the environment-specific details unique to your setup (accounts, host names, API defaults).

## Installing Skills (in container)

```bash
# Install from clawhub registry (persistent to config volume)
docker compose exec --user root openclaw-gateway \
  clawhub install <slug> --workdir /home/node/.openclaw --dir skills

# List installed
docker compose exec openclaw-gateway clawhub list --workdir /home/node/.openclaw --dir skills

# Update all
docker compose exec --user root openclaw-gateway \
  clawhub update --all --workdir /home/node/.openclaw --dir skills
```
