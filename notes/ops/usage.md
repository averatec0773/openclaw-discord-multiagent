# Usage Analysis

Token usage data is stored locally in session JSONL files ‚ÄĒ no external API access required.
This mirrors what the dashboard's Usage page shows.

---

## Data location

Each agent has its own session directory:

```
/root/.openclaw/agents/<agentId>/sessions/     ‚Üź host path (SSH / Claude Code)
/home/node/.openclaw/agents/<agentId>/sessions/ ‚Üź container path (agent's own tools)
‚Ēú‚ĒÄ‚ĒÄ sessions.json          # index: session metadata and state
‚ĒĒ‚ĒÄ‚ĒÄ <session-uuid>.jsonl   # one file per session, line-delimited JSON
```

The agent itself uses the container path. External tools (SSH, Claude Code) use the host path.

Usage data lives inside the JSONL files, in `message.usage` on each assistant turn.

---

## Usage fields per turn

```json
{
  "type": "message",
  "timestamp": "2026-03-11T09:00:00.000Z",
  "message": {
    "role": "assistant",
    "model": "gpt-5.4",
    "usage": {
      "input": 12490,
      "output": 90,
      "cacheRead": 189312,
      "cacheWrite": 0,
      "totalTokens": 201892,
      "cost": {
        "input": 0.031225,
        "output": 0.00135,
        "cacheRead": 0.004733,
        "cacheWrite": 0,
        "total": 0.037308
      }
    }
  }
}
```

| Field | Meaning |
|---|---|
| `input` | New tokens sent to the model this turn (conversation history + new message) |
| `output` | Tokens generated by the model |
| `cacheRead` | Tokens served from the prompt cache ‚ÄĒ much cheaper than `input` |
| `cacheWrite` | Tokens written into the prompt cache (one-time cost) |
| `cost.total` | Actual dollar cost for this turn |

**Cache read vs input:** After the first turn of a session, the system context (AGENTS.md, SOUL.md, etc.) is cached. Subsequent turns show large `cacheRead` and small `input`. Rising `input` with low `cacheRead` means the context has grown past the cache boundary ‚ÄĒ a signal to run `/compact`.

## Analysis script

Run on the server to get a full breakdown by day and model:

```bash
ssh openclaw "cat > /tmp/usage.py << 'PYEOF'
import json, glob

files = glob.glob('/root/.openclaw/agents/main/sessions/*.jsonl')
daily = {}
model_totals = {}
total_cost = 0

for f in sorted(files):
    with open(f) as fh:
        for line in fh:
            try:
                obj = json.loads(line)
                msg = obj.get('message', {})
                usage = msg.get('usage', {})
                ts = obj.get('timestamp', '')[:10]
                if not (usage and (usage.get('input', 0) or usage.get('cacheRead', 0))):
                    continue
                i = usage.get('input', 0)
                o = usage.get('output', 0)
                cr = usage.get('cacheRead', 0)
                cw = usage.get('cacheWrite', 0)
                cost = usage.get('cost', {}).get('total', 0)
                model = msg.get('model', 'unknown')
                total_cost += cost
                if ts:
                    d = daily.setdefault(ts, {'in':0,'out':0,'cr':0,'cw':0,'cost':0,'turns':0})
                    d['in']+=i; d['out']+=o; d['cr']+=cr; d['cw']+=cw
                    d['cost']+=cost; d['turns']+=1
                mt = model_totals.setdefault(model, {'in':0,'out':0,'cr':0,'cost':0})
                mt['in']+=i; mt['out']+=o; mt['cr']+=cr; mt['cost']+=cost
            except:
                pass

print('=== DAILY ===')
for day in sorted(daily):
    d = daily[day]
    tot = d['in']+d['out']+d['cr']+d['cw']
    cpt = d['cost']/d['turns'] if d['turns'] else 0
    print('%s  total=%7.1fK  input=%6.1fK  cacheRead=%6.1fK  cost=\$%.3f  turns=%3d  \$/turn=\$%.4f' % (
        day, tot/1000, d['in']/1000, d['cr']/1000, d['cost'], d['turns'], cpt))

print()
print('=== BY MODEL ===')
for model, m in sorted(model_totals.items(), key=lambda x: -x[1]['cost']):
    print('  %-20s  input=%7.1fK  cacheRead=%7.1fK  cost=\$%.3f' % (
        model, m['in']/1000, m['cr']/1000, m['cost']))

print()
print('TOTAL COST: \$%.4f' % total_cost)
PYEOF
python3 /tmp/usage.py"
```

## Reading the output

**Healthy pattern:**
```
2026-03-30  total=1206.8K  input=469.9K  cacheRead=735.5K  cost=$1.380  turns=9  $/turn=$0.1533
```
- `cacheRead > input` ‚Üí cache is working, context is being reused
- `$/turn` is low and stable

**Unhealthy pattern (context accumulated):**
```
2026-03-13  total=1407.6K  input=1089.6K  cacheRead=312.9K  cost=$2.877  turns=12  $/turn=$0.2398
```
- `input >> cacheRead` ‚Üí cache not helping, large context being resent each turn
- High `$/turn` ‚Üí time to run `/compact`

**Rule of thumb:** if `$/turn` exceeds ~$0.05 on routine conversations, run `/compact`.

## Comparing with the dashboard

The dashboard Usage page pulls from provider APIs and may include usage outside this OpenClaw instance (e.g. other API clients using the same key). The JSONL analysis covers only this instance's sessions.

Expected match:
- `input` total ‚Čą dashboard Input
- `cacheRead` total ‚Čą dashboard Cache Read
- `cacheWrite` total ‚Čą dashboard Cache Write
- `output` total ‚Čą dashboard Output (small discrepancy if public agent sessions exist)

## Per-session context growth check

To see how a single session's context has grown over time:

```bash
ssh openclaw "python3 << 'PYEOF'
import json, glob

# Use the most recently modified session
files = sorted(glob.glob('/root/.openclaw/agents/main/sessions/*.jsonl'),
               key=lambda f: __import__('os').path.getmtime(f))
f = files[-1]
print('Session:', f.split('/')[-1])
with open(f) as fh:
    for line in fh:
        obj = json.loads(line)
        usage = obj.get('message',{}).get('usage',{})
        ts = obj.get('timestamp','')[:10]
        if usage.get('input',0) > 5000:
            total_ctx = usage['input'] + usage.get('cacheRead', 0)
            print('%s  context=%7.1fK  input=%7.1fK  cost=\$%.4f' % (
                ts, total_ctx/1000, usage['input']/1000,
                usage.get('cost',{}).get('total',0)))
PYEOF"
```

Rising `context` numbers across turns confirm context accumulation. See [compaction.md](compaction.md) for how to address it.
